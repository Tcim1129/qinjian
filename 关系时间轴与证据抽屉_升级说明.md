# 关系时间轴与证据抽屉升级说明

## 这次升级做了什么

本轮升级把“关系时间轴回放”从报告页弹窗推进成了独立能力，并补上了“证据抽屉”。

现在系统已经具备：

- 独立的关系时间轴页面
- 时间轴类别筛选与语气筛选
- 七天剧本分支覆盖层
- 时间轴节点选中态
- 事件证据抽屉

## 用户侧体验变化

用户现在可以：

1. 在报告页进入独立时间轴页
2. 看到最近一段关系是如何一步步演化到现在的
3. 按“记录 / 风险 / 行动 / 剧本 / 策略 / 对齐 / 教练”筛选事件
4. 点开任一节点，查看系统为什么记录它
5. 看到该节点会如何影响当前剧本、策略与下一步动作

## 为什么这很重要

以前系统更像“会给结果”。

现在系统开始像“会解释自己”：

- 不只告诉用户结论
- 还能展示结论背后的事件链
- 还能说明某个节点为什么会影响后续决策

这会显著提升三个东西：

- 产品可信度
- 比赛答辩时的系统感
- 后续做可解释 AI 的延展空间

## 当前实现方式

前端层面：

- 报告页增加时间轴入口
- 时间轴页采用“左轨道 + 右解释面板”的结构
- 右侧包括剧本分支覆盖层和事件证据抽屉

后端层面：

- 复用既有 relationship events 数据
- 通过 `/api/v1/insights/timeline` 返回可直接渲染的事件序列
- 时间轴节点当前主要依赖统一事件流中的 payload 做解释

## 下一步最值得做

### 1. 事件详情 API

为单个事件提供更细的证据来源，例如：

- 关联报告摘要
- 关联任务反馈
- 风险变化前后对比
- 关联剧本迁移原因

### 2. 浏览器联调与视觉验收

需要真实打开页面确认：

- 桌面端信息密度是否合适
- 移动端是否出现挤压
- 选中态和证据抽屉是否足够明显

### 3. 比赛展示版时间轴

可进一步做一版更适合答辩展示的模式：

- 默认高亮最近 5 个关键节点
- 自动标出“冲突升级 -> 修复介入 -> 风险回落”的链路
- 展示系统如何从事件流走到策略决策

## 一句话总结

这次升级让项目从“会分析关系”继续走向“会回放关系、解释关系、说明自己为什么这样判断”。
