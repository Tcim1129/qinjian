# 隐私沙盒实施说明

这份说明现在覆盖两轮能力：

- Phase 1：日志脱敏、AI 出口脱敏、签名私有上传
- Phase 2：隐私审计、用户隐私中心、删除请求与保留治理

## 已上线的保护

- 验证码日志脱敏
  - [auth.py](C:/Users/colour/Desktop/亲健/backend/app/api/v1/auth.py) 不再把手机号和验证码原样写入日志。
  - 现在只记录脱敏后的手机号，例如 `138****8000`。

- AI 出口统一脱敏
  - [__init__.py](C:/Users/colour/Desktop/亲健/backend/app/ai/__init__.py) 新增统一的 `create_chat_completion(...)`。
  - 所有走这层的文本请求，都会在发给模型提供商前进行一轮敏感信息清洗。

- 上传文件改为签名私有访问
  - [upload.py](C:/Users/colour/Desktop/亲健/backend/app/api/v1/upload.py) 新增 `GET /api/v1/upload/access/{subdir}/{filename}`。
  - 服务端会返回稳定的存储路径 `url`，同时额外返回带时效签名的 `access_url`。
  - [schemas/__init__.py](C:/Users/colour/Desktop/亲健/backend/app/schemas/__init__.py) 会把数据库里保存的 `/uploads/...` 自动转换成可直接访问的私有地址。
  - [main.py](C:/Users/colour/Desktop/亲健/backend/app/main.py) 默认不再公开挂载整个 `/uploads` 目录，只有显式开启 `UPLOAD_PUBLIC_ACCESS_ENABLED` 才会回到兼容模式。

- 脱敏规则集中管理
  - [privacy_sandbox.py](C:/Users/colour/Desktop/亲健/backend/app/services/privacy_sandbox.py) 统一处理：
    - 手机号
    - 邮箱
    - UUID
    - JWT / token 样式串
    - 超长数字串

- 隐私开关配置化
  - [config.py](C:/Users/colour/Desktop/亲健/backend/app/core/config.py) 新增：
    - `PRIVACY_SANDBOX_ENABLED`
    - `PRIVACY_MASK_LOGS`
    - `PRIVACY_REDACT_LLM_INPUT`

## Phase 2：审计与保留治理

- 隐私审计事件流
  - [privacy_audit.py](C:/Users/colour/Desktop/亲健/backend/app/services/privacy_audit.py) 统一把 AI 文本调用、语音转录、上传创建、状态查看、删除请求、保留清扫写进 `privacy.*` 事件。
  - 审计只存元数据和脱敏摘要，不保存原始 prompt、原始模型输出、原始手机号或 token。

- 用户隐私中心
  - [privacy.py](C:/Users/colour/Desktop/亲健/backend/app/api/v1/privacy.py) 新增：

  - Web 端个人页现在会显示当前保护状态、保留周期、删除请求状态和最近隐私活动。
  - 小程序和 App 个人页新增了简化版隐私卡片。

- 删除请求状态表
  - [__init__.py](C:/Users/colour/Desktop/亲健/backend/app/models/__init__.py) 新增 `PrivacyDeletionRequest`。
  - [2026_03_23_0010_add_privacy_deletion_requests.py](C:/Users/colour/Desktop/亲健/backend/alembic/versions/2026_03_23_0010_add_privacy_deletion_requests.py) 负责建表。

- 保留清扫器

    - 清理 180 天前的 `privacy.*` 审计事件

- 管理后台接口
  - [privacy.py](C:/Users/colour/Desktop/亲健/backend/app/api/v1/admin_routes/privacy.py) 新增：
    - `GET /api/v1/admin/privacy/audits`
    - `GET /api/v1/admin/privacy/delete-requests`
    - `POST /api/v1/admin/privacy/delete-requests/{id}/approve`

## 当前边界

这还不是“零数据出域”方案，目前仍然有几个明确边界：

- 多租户级别的数据隔离策略
- 本地模型 / 端侧推理
- 完整数据导出
- 上传资源的全量作用域感知签名下发

## 当前默认策略

- AI 审计保留：180 天
- 上传签名 URL：60 分钟
- 临时转录残留清理：24 小时
- 用户删除请求宽限期：7 天
- 共享 pair 数据：默认进入人工复核，不自动硬删

## 下一步建议

如果继续向“真实可用的软件”推进，最值得接着补的是：

1. 把上传签名彻底升级成“按当前用户/当前关系归属校验后再签发”。
2. 给保留清扫器接定时任务或后台手工执行入口。
3. 为删除请求补更明确的“导出前确认 / 删除后告知”流程。
