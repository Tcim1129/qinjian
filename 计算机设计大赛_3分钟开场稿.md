# 大学生计算机设计大赛软件赛道 3 分钟开场稿

## 推荐项目名

亲健——面向青年关系场景的关系智能支持系统

## 直接可讲版本

各位评委老师好，我们的参赛作品是《亲健——面向青年关系场景的关系智能支持系统》。

我们关注的是一个很现实、但又很少被软件真正解决的问题：在青年关系场景中，很多矛盾并不是因为重大事件，而是因为日常沟通中的误解积累、情绪表达失真和修复动作缺失。现有同类产品大多停留在记录、聊天或者内容推荐层面，缺少一套能够持续理解关系变化、给出可执行支持，并且能够形成闭环的软件系统。

因此，我们设计并实现了亲健。它不是一个泛聊天机器人，也不是一个单点情感问答工具，而是一个运行在 Web、服务端和数据库之上的关系支持系统。系统围绕“记录、判断、建议、修复、回看”五个环节展开：用户先完成每日记录，系统再结合关系画像和事件流形成当前阶段判断，给出下一步建议；在真实沟通前，用户可以使用消息预演功能降低冲突风险；当关系进入高风险阶段时，系统还可以生成结构化修复协议，并将后续结果继续写回系统。

本项目的重点不是展示某一个模型，而是把关系支持做成一套完整的软件闭环。我们的核心设计包括关系画像、关系事件流、双视角叙事对齐和修复协议等模块。不同模块共享同一套上下文，因此系统不是一次性生成建议，而是能够持续跟踪关系状态、解释判断依据，并支持后续回放和复盘。

在技术实现上，系统采用 Web 前端、FastAPI 后端和 PostgreSQL 数据层构成主体架构，并在此基础上形成记录层、评估层、干预层、修复层和回看层，保证业务数据、状态分析和交互能力能够稳定协同。对计算机设计大赛的软件应用与开发赛道来说，我们希望呈现的不是一个“会回答问题的模型”，而是一套真实可访问、可操作、可演示的 Web 软件系统。

接下来，我们将按照实际使用路径，向各位老师演示亲健如何从一次记录开始，逐步完成关系识别、沟通预演、叙事对齐和冲突修复的完整过程。
