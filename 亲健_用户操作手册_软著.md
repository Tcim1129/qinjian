# 亲健青年亲密关系健康管理平台软件 V1.0

## 用户操作手册

---

### 一、软件概述

**软件名称**：亲健青年亲密关系健康管理平台软件

**版本号**：V1.0

**软件用途**：亲健是中国首个基于AI的亲密关系健康管理平台，面向青年群体（18-35岁），为情侣、夫妻、挚友等亲密关系提供科学的情感评估与个性化改善方案。平台通过每日情感打卡、AI情感分析、关系健康报告和游戏化互动等功能，帮助用户持续提升关系健康度。

**运行环境**：
- 服务器：Ubuntu 22.04+ / 2C4GB以上配置
- 数据库：PostgreSQL 16
- 后端框架：Python 3.12 + FastAPI
- 前端：HTML5 / CSS3 / JavaScript（移动端优先响应式设计）
- AI引擎：硅基流动API（兼容OpenAI格式）
- 容器化：Docker + Docker Compose
- 反向代理：Nginx

**访问方式**：用户通过浏览器访问平台网址即可使用，支持手机和电脑端。

---

### 二、系统架构

亲健平台采用前后端分离的三层架构设计：

```
┌─────────────────────────────────────────┐
│           用户浏览器（前端）              │
│     HTML/CSS/JavaScript 单页应用         │
└──────────────────┬──────────────────────┘
                   │ HTTP/HTTPS
┌──────────────────▼──────────────────────┐
│           Nginx 反向代理                 │
│     静态资源托管 + API请求转发            │
└──────────────────┬──────────────────────┘
                   │
┌──────────────────▼──────────────────────┐
│        FastAPI 后端服务 (Python)          │
│  ┌──────────┐ ┌──────┐ ┌──────────┐    │
│  │ 认证模块  │ │打卡  │ │ AI分析   │    │
│  │ JWT+bcrypt│ │系统  │ │ 情感报告 │    │
│  └──────────┘ └──────┘ └──────────┘    │
│  ┌──────────┐ ┌──────┐ ┌──────────┐    │
│  │ 配对系统  │ │报告  │ │ 关系树   │    │
│  │ 邀请码   │ │生成  │ │ 游戏化   │    │
│  └──────────┘ └──────┘ └──────────┘    │
└──────────────────┬──────────────────────┘
         ┌─────────┴─────────┐
┌────────▼────────┐ ┌───────▼────────┐
│  PostgreSQL 16   │ │  硅基流动 AI    │
│  关系型数据库     │ │ DeepSeek/GPT-4o│
└─────────────────┘ └────────────────┘
```

**核心技术特性**：
- 异步高性能：基于FastAPI的全异步架构
- 隐私沙盒：对方无法查看原始打卡内容，仅AI分析后的洞察可见
- 循证心理学：基于戈特曼理论和依恋理论的科学分析框架
- 游戏化设计：关系树成长系统增强用户粘性

### 三、安装与部署

#### 3.1 环境要求

- 操作系统：Ubuntu 22.04+ 或 Debian 12+
- 硬件配置：2核CPU / 4GB内存 / 20GB存储
- 软件依赖：Docker、Docker Compose、Git

#### 3.2 一键部署

```bash
# 1. 克隆代码仓库
git clone https://github.com/Tcim1129/qinjian.git
cd qinjian

# 2. 运行部署脚本
sudo bash deploy.sh
```

部署脚本将自动完成以下操作：
1. 检测并安装 Docker 和 Docker Compose
2. 生成安全密钥和数据库密码
3. 提示输入 AI 服务 API Key
4. 构建 Docker 镜像并启动全部服务
5. 执行健康检查，确认服务正常运行

#### 3.3 服务管理

| 操作 | 命令 |
|------|------|
| 查看日志 | `docker compose logs -f` |
| 重启服务 | `docker compose restart` |
| 停止服务 | `docker compose down` |
| 更新代码 | `git pull && docker compose up -d --build` |

### 四、功能操作说明

#### 4.1 用户注册

首次使用亲健平台，用户需要注册一个新账号。

**操作步骤**：
1. 打开浏览器，访问亲健平台地址
2. 在登录页面底部，点击「立即注册」链接
3. 填写以下信息：
   - **昵称**：您希望对方称呼您的名字
   - **邮箱**：用于登录的电子邮箱地址
   - **密码**：至少6位的登录密码
4. 点击「注册」按钮完成注册

注册成功后，系统将自动登录并跳转至配对页面。

#### 4.3 创建配对

登录后，如果尚未建立配对关系，系统将引导用户创建配对。

**操作步骤**：
1. 选择关系类型（三选一）：
   - 💕 **情侣**：恋爱关系
   - 💍 **夫妻**：婚姻关系
   - 🤜🤛 **挚友**：亲密友谊
2. 点击「创建配对」按钮
3. 系统生成一个 **8位邀请码**
4. 将邀请码发送给对方

**邀请码机制**：
- 每个配对关系对应一个唯一的8位字母数字邀请码
- 对方在自己的账号中输入邀请码即可完成配对
- 每个用户同一时间只能拥有一个活跃的配对关系

#### 4.4 加入配对

收到对方邀请码的用户可以加入已创建的配对。

**操作步骤**：
1. 在配对页面下方，找到「加入已有配对」区域
2. 输入对方提供的8位邀请码
3. 点击「加入配对」按钮

配对成功后，双方即可开始使用打卡和报告等全部功能。

#### 4.5 每日情感打卡

打卡是亲健平台的核心功能，用户每天记录自己的情感状态和与对方的互动感受。

**操作步骤**：
1. 点击底部导航栏的「✍️ 打卡」标签
2. 在文本框中输入今天的感受和发生的事情
3. （可选）选择今天的情绪标签：开心、平静、感动、期待、焦虑、委屈、生气、疲惫
4. （可选）点击「📷 添加图片」上传照片
5. （可选）点击「🎤 添加语音」上传语音记录
6. 点击「提交打卡」按钮

**打卡后的自动处理**：
- 系统自动调用AI对打卡内容进行情感分析，评估情绪得分
- 如果**仅一方完成打卡**：系统自动生成个人情感日记（Solo Report）
- 如果**双方都完成打卡**：系统自动生成每日关系健康报告（Daily Report）

**隐私保护**：
- 打卡的原始文字内容仅本人和AI可见，对方**无法查看**原文
- 对方只能看到AI分析后的洞察和建议，不包含具体内容

---

#### 4.6 首页状态查看

首页展示当日的核心状态信息。

**页面内容**：
- **连续打卡天数**：显示用户的连续打卡记录（用火焰图标标识）
- **今日打卡状态卡片**：
  - 显示自己和对方的打卡状态（已打卡/未打卡/等待中）
  - 未打卡时可点击「开始打卡」按钮快速进入打卡页面
- **关系树**：显示当前关系树的成长状态（详见4.8节）
- **报告入口**：
  - 双方都完成打卡后，显示「查看今日报告」按钮
  - 仅单方完成时，显示「查看个人情感日记」按钮

#### 4.7 AI 关系健康报告

亲健平台的AI报告系统基于循证心理学理论框架，提供三个层次的分析报告。

##### 4.7.1 每日报告

**生成条件**：双方均完成当日打卡后自动生成

**报告内容**：
- **健康指数**：1-100分的关系健康度可视化仪表盘
- **双方情绪分析**：A方和B方各自的情绪得分（1-10）和情绪标签
- **沟通质量评价**：基于戈特曼「5:1积极互动比」理论的沟通质量评分
- **今日亮点**：AI识别的关系中积极的互动信号
- **改善建议**：一条具体可执行的关系改善建议
- **风险信号**：基于戈特曼「末日四骑士」模型的预警（如有）
- **理论标注**：标明本次分析主要参考的心理学理论

##### 4.7.2 周报

**生成条件**：至少累计3天的完整日报数据

**操作方式**：在报告页面点击「📋 生成周报」按钮

**报告内容**：
- 周健康指数与趋势（上升/稳定/下降）
- 双方情绪趋势图
- 沟通模式分析
- 本周亮点汇总
- 下周行动建议

##### 4.7.3 月报

**生成条件**：至少累计2周的完整数据

**操作方式**：在报告页面点击「📑 生成月报」按钮

**报告内容**：
- 月度健康指数与长期趋势分析
- 双方情绪模式深度解读（基于依恋理论）
- 互动模式分析（参考末日四骑士模型）
- 关系优势与成长空间
- 月度里程碑回顾
- 下月改善目标
- 专业建议（如发现高风险信号，温和提醒是否需寻求专业咨询）

##### 4.7.4 健康趋势图

在报告页面下方，系统自动绘制近14天的健康度趋势折线图，并标注趋势方向：
- 📈 **持续上升**：关系健康度在改善
- ➡️ **保持稳定**：关系状态平稳
- 📉 **需要关注**：关系健康度有所下降

#### 4.8 关系树（游戏化系统）

关系树是亲健平台的游戏化核心，通过「养成」一棵虚拟树来象征关系的成长。

**成长等级**：

| 等级 | 名称 | 图标 | 所需成长值 |
|------|------|------|-----------|
| 1 | 种子 | 🌰 | 0 |
| 2 | 嫩芽 | 🌱 | 50 |
| 3 | 树苗 | 🌿 | 150 |
| 4 | 小树 | 🌳 | 350 |
| 5 | 花开 | 🌸 | 700 |

**获取成长值的方式**：

| 行为 | 成长值 |
|------|--------|
| 单方打卡 | +10 |
| 双方都完成打卡（额外奖励） | +15 |
| 连续打卡7天以上（额外奖励） | +10 |
| 手动浇水（每日限一次） | +5 |

**操作方式**：
- 在首页可以看到关系树的当前状态、成长值和进度条
- 点击「💧 浇水」按钮可每日为关系树浇水一次
- 升级时系统会记录里程碑事件

### 五、隐私与安全设计

亲健平台高度重视用户隐私和数据安全，主要保护措施如下：

| 安全措施 | 具体实现 |
|---------|---------|
| 用户认证 | JWT（JSON Web Token）令牌认证，有效期7天 |
| 密码存储 | bcrypt 单向哈希算法，不存储明文密码 |
| 隐私沙盒 | 打卡原始内容仅本人可见，对方仅能看到AI分析结果 |
| 文件上传 | 严格的文件类型和大小限制（最大10MB），流式写入防止OOM |
| API 安全 | 所有涉及用户数据的接口均需携带有效的认证令牌 |
| 数据隔离 | 用户只能访问自己所属配对关系的数据 |

### 六、常见问题

**Q1: 忘记密码怎么办？**
A: 当前版本暂不支持自助找回密码功能，请联系管理员重置。

**Q2: 可以同时和多个人配对吗？**
A: 不可以。每个用户同一时间只能拥有一个活跃的配对关系。

**Q3: 对方能看到我写的打卡内容吗？**
A: 不能。您的原始打卡内容属于隐私保护范围，对方只能看到AI分析后的洞察报告。

**Q4: AI报告需要等多久？**
A: 通常在提交打卡后10-60秒内生成。页面会自动轮询检测报告状态，生成完毕后会有提示。

**Q5: 支持哪些文件格式上传？**
A: 图片支持 JPEG、PNG、WebP、GIF 格式；语音支持 MP3、WAV、OGG、WebM、MP4、AAC 格式。单文件最大10MB。
