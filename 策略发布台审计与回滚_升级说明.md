# 策略发布台审计与回滚 升级说明

## 这次补了什么

这轮把管理员策略发布台从“可编辑”升级成了“可发布、可追踪、可回滚”。

- 后端复用了现有 `RelationshipEvent` 作为管理员发布审计日志
- 每次创建、编辑、启停、排序、回滚都会生成一条 `admin.policy.*` 事件
- 每条审计事件都保存了：
  - 操作者
  - 操作类型
  - 策略前后快照
  - 字段级变化摘要
  - 可选备注和额外元信息

## 新增接口

- `GET /api/v1/admin/policies/{policy_id}/audit`
  - 返回这条策略最近的管理员发布历史
  - 默认按时间倒序

- `POST /api/v1/admin/policies/{policy_id}/rollback`
  - 传入目标审计事件 ID
  - 系统会把策略恢复到该事件对应的 `after` 快照
  - 同时追加一条新的 `admin.policy.rolled_back` 事件

## 前端表现

- 策略发布台卡片新增“审计”按钮
- 审计页会展示：
  - 最近动作
  - 操作者
  - 字段级前后变化
  - 一键回滚按钮

## 设计取舍

- 没有新建策略审计表，而是沿用事件流
  - 好处是接入成本低，和现有时间轴/审计体系一致
  - 也方便后面把发布动作继续并入更大的运营审计中心

- 回滚不是覆盖历史，而是追加一条新事件
  - 这样发布历史始终完整
  - 对比赛答辩和后续运营都更容易解释
