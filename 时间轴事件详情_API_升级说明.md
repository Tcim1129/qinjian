# 时间轴事件详情 API 升级说明

## 这次补了什么

本轮把“时间轴证据抽屉”从静态说明卡，升级成了真正会按节点拉详情的证据面板。

- 新增接口：`GET /api/v1/insights/timeline/events/{event_id}`
- 返回内容：事件基础信息、指标列表、证据卡片、当前上下文
- 前端行为：点击时间轴节点后，右侧抽屉先进入加载态，再切到详情态
- 容错策略：如果详情请求失败，抽屉仍会退回到本地 payload fallback，不会整块空掉

## 为什么值得做

- 让时间轴从“能回放”升级成“能解释”
- 每个节点都能说明自己为什么重要、会影响什么
- 后续做审计日志、答辩演示、可解释 AI 时，这层可以直接复用

## 当前实现范围

后端：

- `backend/app/api/v1/insights.py`
- `backend/app/schemas/__init__.py`

前端：

- `web/js/api.js`
- `web/js/app.js`
- `web/css/style.css`

## 后续最值的两步

1. 把事件详情继续接到真实关联数据上，例如完整报告摘要、任务反馈明细、危机前后对比。
2. 给策略后台补审计日志，让“为什么这次切了策略”能和时间轴节点互相跳转。
